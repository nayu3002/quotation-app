# Quotation SaaS Platform

A full-stack, monorepo-style project for a Quotation Software-as-a-Service (SaaS) platform, featuring both a Web Application and a Mobile Application.

This platform allows users to manage, generate, and track quotations seamlessly across web and mobile platforms.

## 📁 Repository Structure

- `quotation app/` - The web application built with Next.js.
- `mobile/` - The mobile application built with React Native and Expo.

---

## 💻 Web Application (`quotation app`)

The web frontend is a modern, responsive dashboard built to handle complex quotation logic, user management, and PDF generation.

### 📂 Folder Structure (`quotation app/src/`)
- `app/` - Next.js App Router root.
  - `(auth)/` - Authentication routes (`login`, `register`).
  - `(dashboard)/` - Main dashboard views (`cost-templates`, `customers`, `products`, `quotations`, `settings`).
  - `api/` - Backend API routes.
- `components/` - Reusable UI components (e.g., `sidebar`, `import-mapper`).
- `lib/` - Utilities and configuration.
  - `supabase/` - Supabase client initialization.
  - `calc.ts` - Quotation calculation logic.
  - `prisma.ts` - Prisma ORM setup.
  - `org-context.ts` - Organization state context.

### Tech Stack
- **Framework:** Next.js 14 (App Router)
- **Language:** TypeScript
- **Styling:** Tailwind CSS, PostCSS
- **State Management:** Zustand
- **Database ORM:** Prisma
- **Authentication/Backend:** Supabase & Supabase SSR
- **Forms & Validation:** React Hook Form + Zod
- **PDF Generation:** `@react-pdf/renderer`, `html2pdf.js`
- **UI Components:** `lucide-react`, `clsx`, `tailwind-merge`, `sonner` (Toast notifications)

### Getting Started

1. Navigate to the web app directory:
   ```bash
   cd "quotation app"
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Set up your environment variables:
   Copy `.env.example` to `.env` and fill in your Supabase and Database credentials.

4. Generate Prisma Client:
   ```bash
   npm run postinstall
   # or
   npm run db:generate
   ```

5. Push the database schema:
   ```bash
   npm run db:push
   ```

6. Start the development server:
   ```bash
   npm run dev
   ```
   The app will be available on all network interfaces (`-H 0.0.0.0`).

### Available Scripts

- `npm run dev`: Starts the Next.js development server.
- `npm run build`: Builds the app for production.
- `npm run start`: Starts the production server.
- `npm run lint`: Runs ESLint for code formatting.
- `npm run db:push`: Pushes schema changes to the connected database.
- `npm run db:generate`: Generates Prisma Client.
- `npm run db:studio`: Opens Prisma Studio to view and edit database records.

## 📱 Mobile Application (`mobile`)

The mobile companion app allows users to access their quotations on the go, native to iOS and Android.

### 📂 Folder Structure (`mobile/`)
- `app/` - Expo Router root directory.
  - `(auth)/` - Authentication screens (`login.tsx`).
  - `(tabs)/` - Main tab-based navigation screens (`cost-templates`, `create-quotation`, `customers`, `products`, `quotations`, `settings`).
- `lib/` - Shared utilities and configurations (`supabase.ts`, `config.ts`).
- `store/` - Zustand global state management (`useAppStore.ts`).
- `assets/` - Images, fonts, and static resources.
- `android/` & `ios/` - Native platform code (generated by Expo).

### Tech Stack
- **Framework:** Expo & React Native (Expo Router)
- **Language:** TypeScript
- **Styling:** NativeWind (Tailwind CSS for React Native)
- **State Management:** Zustand
- **Data Fetching:** `@tanstack/react-query`
- **Authentication/Backend:** Supabase (`@supabase/supabase-js`)
- **Storage:** `@react-native-async-storage/async-storage`
- **Icons:** `lucide-react-native`, `@expo/vector-icons`

### Getting Started

1. Navigate to the mobile app directory:
   ```bash
   cd mobile
   ```

2. Install dependencies:
   ```bash
   npm install
   ```

3. Set up environment variables in a `.env` file (ensure you include Supabase URL and Anon Key).

4. Start the Expo development server:
   ```bash
   npm start
   ```

5. Run on your preferred platform:
   - Press `a` for Android emulator.
   - Press `i` for iOS simulator.
   - Scan the QR code with the Expo Go app on a physical device.

### Available Scripts

- `npm start`: Starts the Expo Metro bundler.
- `npm run android`: Runs the app natively on Android.
- `npm run ios`: Runs the app natively on iOS.
- `npm run web`: Starts the app in a web browser using React Native Web.

## 🛠 Backend Infrastructure

Both applications share a backend powered by **Supabase**.
- **Database:** PostgreSQL (managed by Supabase and structured via Prisma).
- **Authentication:** Supabase Auth for cross-platform secure login.
- **Storage:** Supabase Storage (if applicable, for storing PDF assets or user avatars).
